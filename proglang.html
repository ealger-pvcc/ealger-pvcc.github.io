Chapter 6 Terminology: Programming Languages

Term Definition
Machine Language Programs for modern computers that consist of sequences of instructions that are encoded as numeric digits
Debugging Errors that must be located and corrected
Assembler Converts mnemonic expressions into machine language instructions
Assembly Language The mnemonic system for representing programs
Translator Translates programs expressed in high-level primitives into machine-language programs
Compiler Compiles several machine instructions into short sequences to stimulate the activity requested by a single high-level primitive
Interpreter An alternative to translators, another means of implementing third-generation languages; executes the instructions as they are translated
Cross-platform Software Software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network
Natural Language English, German, and Latin
Formal Language Programming languages
Machine Independence Program language that does not rely on the characteristics of a particular machine
High-level Programming Language A more user-friendly level of programming language that enables construction of increasingly complex software designs
Object-oriented Programming (OOP) A software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects
Scripting Language Used to perform administrative tasks rather than to develop complex programs; the expression of such task is known as a script
Data Types:
    1.Integer
    2.Float
    3.String
    1. The numeric data consisting of whole numbers, probably stored using two's complement notation
    2. The numeric data that might contain values other that whole numbers, probably stored in floating-point notation
    3. A sequence of data
Primitive Data Type The data types that are included as primitives in a programming language, such as int for integer and char for character
Data Structure The conceptual shape or arrangement of data
Python Collection Data Structures:
    1.List
    2.Tuple
    3.Set
    4.Dictionary
    1.Created using square brackets, lists are used to store multiple items in a single variable
    2.Written with round brackets; a tuple is a collection which is ordered and unchangeable
    3.Written with curly brackets; a set is a collection which is unordered, unchangeable, and unindexed
    4.Written with curly brackets; a dictionary is a collection which is ordered, changeable, and do not allow duplicates
Function A set of instructions for performing a task that can be used as an abstract tool by other program units
Local Variable A variable declared within a function; it can be referenced only within that function
Global Variable Variables whose scopes are not restricted to a particular part of a program
Source Program A program in its original form
Object Program The translated version of a program
