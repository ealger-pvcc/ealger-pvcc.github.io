Operating Systems Terminology

Term Definition
Operating system The software that controls the overall operation of the computer including file management, the user interface, and the program execution environment.
Job queue In batch processing systems, job queue refers to the jobs residing in mass storage waiting for execution.
FIFO The way in which a queue is ordered: "first-in, first-out"; the jobs are removed from the queue in the order in which they arrived.
Interactive processing An operating system developed to allow a program being executed to carry on a dialogue with the user through remote terminals.
Time-sharing An operating system that provided service to multiple users at the same time.
Multiprogramming Time is divided into intervals and then the execution of each job is restricted to only one interval at a time. At the end of each interval, the current job is temporarily set aside and another is allowed to execute during the next interval in rapid succession.
Application Software Consists of the programs for performing tasks particular to the machine's utilization.
System Software performs tasks that are common to computer systems in general; the infrastructure that the application software requires.
Utility Software Consists of programs for performing activities that are fundamental to computer installations but not included in the operating system.
Graphical User Interface (GUI) Objects, such as files and programs, can be manipulated and represented pictorially on the display as icons.
Shell Older user interfaces.
Kernel The internal part of the operating system. Contains software components that perform the very basic functions required by the computer installation.
Device Driver A component of the kernel; the software units that communicate with the controllers to carry out operations on the peripheral devices attached to the machine.
File Manager Coordinates the use of the machine's mass storage facilities.
Memory Manager A component of the kernel; has the task of coordinating the machine's use of main memory.
Virtual memory The large "fictional" memory space created by paging.
Boot strapping A procedure performed by a computer each time it is turned on; transfers the operating system from mass storage to into main memory.
ROM "Read only memory"; a small portion of a computer's main memory where the CPU expects to find its initial program is constructed from special nonvolatile memory cells, the contents can be read but not altered.
Deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
Spooling The technique of holding data for output at a later but more convenient time.
Super user (or administrator) A person who gains highly privileged access to the operating system by identifying themselves as the administrator (usually by name and password) during the login procedure.
Multi-factor authentication A way to solve the pervasive problem of humans using weak passwords: secures one's computer account by requiring more than one piece of evidence to gain access.
