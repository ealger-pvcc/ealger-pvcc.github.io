Chapter 7 Terminology: Software Engineering

Term Definition
Software Engineering The branch of computer science that deals with the design, testing, development, and maintenance of software applications.
Integrated development environment (IDE) Sophisticated packages designed primarily for the software engineering environment; combines tools for developing software into one integrated package.
Association for Computing Machinary (ACM)
    Purpose of ACM: The worlds largest computing society which works to bring people together to support technical excellence, promote lifelong learning and innovation, and recognizing the diverse population of innovators.
    Benefits of ACM membership: An ACM membership includes access to education, volunteer, and job opportunities as well as a sense of community in the world of computing.
    Student ACM membership: A student member enjoys most all of the benefits of a member, with a few exceptions.
Institute of Electrical and Electronics Engineers (IEEE)
    Purpose of IEEE: According to the IEEE website, the organization strives to foster technological advancements for the benefits to society. IEEE provides educational opportunities such as conferences and published material.
    Benefits of IEEE membership: Collaborate with others, network, further your education, and attend IEEE activities and events.
    Student IEEE membership: The benfits of a student membership include: student discount after graduation, prestige, and resources entering the workforce.
Software Life Cycle: development phase
    1. Requirements analysis
    2. Design
    3. Implementation
    4. Testing
    1. Specifies what services the proposed system will provide, to identify any conditions on those services and to define how the outside world will interact with the system.
    2. Creating a plan for the construction of the proposed system; developing a solution to a problem is the goal of design.
    3. The actual writing of programs, creation of data files, and development of databases.
    4. One segement in the overall struggle for quality assurance, whcih is an objective that permeates the entire software life cycle.
Modularity The division of software into manageable units called modules, each of which deals with only a part of the software's overall responsibility.
Interpreter Translates source code into object code one instruction at a time.
Unified Modeling Language (UML) The more modern collection of tools, with the object-oriented paradigm in mind.
Use case Interactions between the system and its users are represented as ovals.
Use case diagram A tool within UML which captures the image of the proposed system from the user's point of view.
Data Relationships
    1. One-to-one
    2. One-to-many
    3. Many-to-many
    1. The association of one item (x) to one (y).
    2. One (x) to many entities of (y).
    3. Many entities of (x) assciated with many entities of (y).
Design pattern A predeveloped model for solving a recurring problem in a software design.
Software quality assurance (SQA) Groups which are charged with overseeing and enforcing the quality control systems adopted by an organization.
Software testing
    1. Glass box testing
    2. Black box testing
    3. Beta Testing
    4. Alpha testing
    1. The software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
    2. Tests that don't rely on knowledge of the software's interior composition.
    3. Falls within the black-box category; a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real life situations before the final version is solidified and released to the public.
    4. Similar to beta testing performed at the developer's site.
Documentation
    1. User documentation
    2. System documentation
    3. Technical Documentation
    1. Explains the features of the software and describes how to use them; meant to be read by the user.
    2. Descibes the software's internal composition so that the software can be maintained later in its life cycle.
    3. Describes how a software system shouold be installed and serviced (such as adjusting operating perameters, installing updates, and reporting problems back to the software's developer).
Intellectual Property Law
    1. Software license
    2. Software disclaimer
    3. Best defenses against software liability claims
    1. A legal agreement between the owner and the user of a software product that grants the user certain permissions to use the product without transferring ownership rights to the intellectual property.
    2. States the limitations of liability.
    3. Applying sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors.
